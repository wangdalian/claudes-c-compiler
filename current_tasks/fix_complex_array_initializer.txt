Fix complex array initializer lowering

Problem: Arrays of complex types (e.g., double _Complex arr[2] = {1.0 + 2.0i, 3.0 + 4.0i})
initialize to all zeros instead of the correct values. This affects ~675 test failures (83%
of all failing tests).

Root cause: Both global and local complex array initialization paths don't handle complex
element types properly:
1. Global: eval_const_expr doesn't understand imaginary literals, falls back to zero
2. Local: stores the pointer to the complex temp, not the actual {real,imag} data

Fix:
- global_init.rs: detect complex array elements and use eval_complex_global_init per element
- stmt.rs: detect complex array elements and use Memcpy instead of scalar store
