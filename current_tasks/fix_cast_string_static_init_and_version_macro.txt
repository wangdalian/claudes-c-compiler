Task: Fix cast-of-string-literal in static initializers and add __VERSION__ macro

Issues:
1. Static initializers like `(unsigned char *)"hello"` emit NULL pointers
   instead of string literal addresses. The eval_string_literal_addr_expr
   function handles casts (recursing into inner expression) but was missing
   the base case for bare StringLiteral/WideStringLiteral/Char16StringLiteral.

2. __VERSION__ predefined macro was not defined, causing coremark to fail
   to parse (it uses "GCC"__VERSION__ string concatenation).

Fix: Add StringLiteral/WideStringLiteral/Char16StringLiteral cases to
eval_string_literal_addr_expr, and define __VERSION__ as "6.5.0".

Status: in_progress
