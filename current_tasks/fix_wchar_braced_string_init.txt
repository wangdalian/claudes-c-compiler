Fix wchar_t array initialization with braced wide string literal (in_progress)

Bug: `wchar_t w[] = {L"hello"};` (braces around wide string literal) stores a pointer
value instead of copying the string contents into the array.

Root cause: In stmt_init.rs lower_1d_array_init, there's a special case for char arrays
with StringLiteral wrapped in braces (lines ~903-910), but no corresponding case for
wchar_t arrays with WideStringLiteral.

The fix needs to add a check for WideStringLiteral in lower_1d_array_init (stmt_init.rs)
and for global init (global_init.rs), similar to the existing StringLiteral handling.

Files to modify:
- src/ir/lowering/stmt_init.rs (local init)
- src/ir/lowering/global_init.rs (global init)

Tests affected:
- compiler_suite_0172_0027 (x86)
- compiler_suite_0172_0032 (x86)
- compiler_suite_0151_0080 (x86, wchar string tests)
