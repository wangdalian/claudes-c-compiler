Task: Fix __int128 <-> float/double conversion (in_progress)

Bug: i128/u128 to float/double casts produce incorrect results (only
low 64 bits were used). Similarly, float/double to i128/u128 casts
did not handle float source types (treated them as integer widening).

Impact: Causes PostgreSQL numeric tests to hang (infinite loop in
ln_var -> sqrt_var due to wrong intermediate results from __int128
to double conversion).

Fix: Added compiler-rt runtime calls for i128 <-> float conversions:
- i128 -> f64: __floattidf / __floatuntidf
- i128 -> f32: __floattisf / __floatuntisf
- f64 -> i128: __fixdfti / __fixunsdfti
- f32 -> i128: __fixsfti / __fixunssfti

Changes:
- src/backend/traits.rs: Added float type checks in emit_cast_default
  before the integer widening/narrowing paths. Added two new trait
  methods: emit_i128_to_float_call, emit_float_to_i128_call.
- src/backend/x86/codegen/codegen.rs: Implemented for SysV x86-64 ABI.
- src/backend/arm/codegen/codegen.rs: Implemented for AAPCS64 ABI.
- src/backend/riscv/codegen/codegen.rs: Implemented for LP64D ABI.
- tests/int128-float-conversion/: New regression test.
