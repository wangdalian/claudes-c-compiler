Deduplicate ARM/RISC-V F128 Soft-Float Code
=============================================
Priority: MEDIUM
Estimated duplicate lines: ~595

Problem
-------
ARM (f128.rs: 337 lines) and RISC-V (f128.rs: 316 lines) backends both
implement IEEE binary128 via identical soft-float library call patterns.
The algorithmic structure is the same, differing only in register names
and instruction syntax:

- ARM: Q registers (q0/q1), x0:x1 GP pairs, "bl" calls, "ldr"/"str"
- RISC-V: a0:a1/a2:a3 GP pairs, "call" instruction, "ld"/"sd"

Functions with near-identical logic across both backends:
1. emit_f128_operand_to_{q0_full,a0_a1} - load f128 from tracked source
2. emit_f128_store_to_slot - store f128 to stack slot
3. emit_f128_store_to_addr_in_{x17,t5} - store f128 to address register
4. emit_f128_neg_full - negate f128 via sign bit flip
5. emit_f128_binop_softfloat - f128 arithmetic via libcalls
6. emit_f128_{cmp,load,cast} in codegen.rs - additional patterns

Additionally, in codegen.rs:
- emit_cast_instrs: ARM/RISC-V both match on CastKind identically
- emit_f128_cmp: same libcall orchestration pattern
- emit_float_binop (F128 path): same libcall pattern
- emit_i128_shl/lshr/ashr (6 functions): identical algorithm

Approach
--------
Create a shared module (e.g., backend/f128_softfloat.rs) that provides
the common soft-float orchestration, parameterized by a small trait or
struct defining arch-specific primitives:
- GP pair register names for arg passing
- How to load/store 16 bytes from a slot
- How to call a function
- How to move between GP pair and f128 storage

Each backend would implement ~5 small primitives and delegate the
orchestration to shared code.

Benefit: Remove ~300-400 lines of structural duplication and ensure
both backends stay consistent when F128 handling is improved.

Key files:
- src/backend/arm/codegen/f128.rs
- src/backend/riscv/codegen/f128.rs
- src/backend/arm/codegen/codegen.rs (emit_cast_instrs, emit_f128_cmp, etc.)
- src/backend/riscv/codegen/codegen.rs (same functions)
- src/backend/cast.rs (shared classification, already extracted)
